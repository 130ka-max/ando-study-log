# このプロジェクトの学習観・教育観

このドキュメントは、Codex Study Harness の各ファイル（`README.md`、`AGENTS.md`、
`prompts/`、`docs/`）に埋め込まれた設計思想を読み取り、その背景にある学習観・
教育観を言語化したものです。設計判断に迷ったときの拠りどころとして使います。

## 1. 中心にある一文

> 大事なのは、AIに答えだけを作ってもらうことではありません。
> 自分が何を疑問に思い、どう予想し、何を確かめ、どう考え直したかを残すことです。

このプロジェクトのあらゆる仕組みは、この一文を実装したものと言えます。
AIは答えの供給源ではなく、思考の伴走者として位置づけられています。

## 2. 学習観

### 2.1 学びとは「答えの獲得」ではなく「見方の変化」である

評価の対象は正答そのものではなく、思考が動いた軌跡です。
ハーネスは一貫して次のループを残すことを求めます。

```text
問い -> 予想 -> 確認 -> 考え直し -> 気づき -> 次に使う
```

公開されるHTMLが「点数表ではなく、考えが深まった順番を見せるページ」と
明言されているのは、この学習観の直接の表れです。
「最初はAと考えたが、Bを確認してCと考えるようになった」という
**見方の更新そのもの**を学習成果として扱います。

### 2.2 知識は「事実の蓄積」ではなく「使える判断基準」になって初めて価値を持つ

`insight-capture-checklist.md` は、事実メモ（弱い記録）と知見ログ（よい記録）を
明確に区別します。「何を調べたか」で止まらず、「なぜその見方の方が筋がよいと
わかったか」「次に似た問題を見たら何から確認するか」まで書けて初めて
学習が成立したとみなす、という立場です。

### 2.3 間違いは矯正対象ではなく、最良の学習素材である

`mistake-analyzer.md` の目的は「正解を知ること」を明確に否定し、
「なぜ間違えたか」「どこまでは正しかったか」「どこでずれたか」を
分解することに置かれています。間違いを原因の種類（知識不足／読み落とし／
条件整理／計算ミス／用語理解／解法選択／時間配分）で分類し、
再発を防ぐ見分け方（チェックポイント）に変換します。
間違いは「次に同じ型を見抜くためのセンサー」を作る機会だという考え方です。

### 2.4 不確かさは隠さず、不確かなまま正しく残す

「未確認」「要確認」を明示するルール、根拠のない断定の禁止、
出典が必要な内容での確認なし断言の禁止——これらは、
**わからないことを正直に保留する力**も学習能力の一部とみなす姿勢です。
仮説は仮説として、未検証は未検証として記録に残します。

### 2.5 学びは一度きりではなく、復習で閉じる

次回復習の内容・日付が決まるまでIssueをCloseしない、という完了条件は、
学習を「その場の理解」ではなく「時間をまたいで定着させる営み」と
捉えていることを示します。学習は次の学習につながって初めて終わります。

## 3. 教育観（AI＝Codexの役割設計）

### 3.1 答えの前に、必ず学習者の現在地を確認する

ハーネス開始時の確認項目（今わかっていること／わからないこと／
まず自分で考えたこと）も、`socratic-tutor.md` の冒頭の問い
（「まず、どこまで自分で考えましたか？」）も、
**学習者の思考から出発する**という原則の徹底です。

### 3.2 足場かけ（スキャフォルディング）を、出しすぎない

- 一度の返信で問い・ヒントは最大2つまで。
- 問題を小さなステップに分ける。
- 学習者が答えたら、まずどこまで正しいかを確認してから次を出す。

支援は「与える」ものではなく「学習者が自分で進めるための最小の足場」
として設計されています。先回りして仮説を作らない（学習者の予想が
未確認なら、Codexが代わりに作らず本人に問い返す）という規定は、
この原則の最も厳格な適用です。

### 3.3 問いの質を上げる

「なぜそう思った？」で終わらせず、「どこを見てそう思った？」
「最初の考えからどこを直す？」まで踏み込む。
根拠と修正点に踏み込む問いを重ねることで、メタ認知
（自分の思考を観察し調整する力）を育てることを狙っています。

### 3.4 学習者自身の言葉で説明できることをゴールにする

完了条件の筆頭は「何を学んだかを自分の言葉で説明できる」。
Close前チェックも穴埋め3文（最初は＿＿＿と考えていた／しかし＿＿＿を
確認して＿＿＿と考えるようになった／次は＿＿＿を確認する）で、
**自分の言葉での再構成**を要求します。理解とは説明できることだ、
という立場です。

### 3.5 やってはいけないことを明示する（教育的禁止事項）

`AGENTS.md` の禁止事項は、教育観のネガフィルムです。

- 学習者が考える前に最終解答だけを提示しない。
- 宿題・レポートを本人の思考なしに完成させない。
- 間違いをただ訂正するだけで原因分析をしない。
- 根拠のない知識を断定しない。
- 学習者の弱点や次回復習を記録せずに終えない。

これらは「AIに任せると学びが空洞化する典型パターン」を
あらかじめ封じる設計です。

## 4. 「記録する」こと自体を学習行為とみなす

このプロジェクトの特徴は、学習プロセスを GitHub Issue・学習ログ・
ポートフォリオという外部化された形に必ず残させる点です。

- Issueは「1つの学習カード」。あとから読めば思考の深まり方がわかる。
- ラベル整理を開始時の必須処理にし、学習を分類可能・検索可能にする。
- 思考ループの1周目を開始直後から可視化する（開始時一周記録）。

記録は学習の副産物ではなく、**思考を外在化して見直すための学習装置**
であり、同時に他者（教師・将来の自分・リポジトリをコピーする他者）が
学びの過程を追体験できる資産でもあります。

## 5. 公開と言語に関する価値観

- 公開HTMLは成果の誇示ではなく、思考の道すじの共有。
  個人情報や未整理物は `portfolio:hide` で守る（公開＝善ではない）。
- ラベル名は英語（機械可読性・再利用性のため）、説明文は
  中学生にもわかる日本語。**仕組みの堅牢さと、学習者へのやさしさ**を
  両立させる設計判断。
- 振り返りと理解確認は、英語学習であっても原則日本語で残す。
  メタ認知は母語で行う方が深まる、という判断。

## 6. 一言でいうと

> このプロジェクトは「AIを使うと学びが速くなる」ではなく、
> 「AIを使っても学びを空洞化させない」ための装置である。
>
> 学習者を、答えの消費者ではなく、
> 自分の思考の変化を観察し説明できる人に育てることを目的としている。
